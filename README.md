# DevOps СИТУ Лабораторная работа №4: Настройка виртуальных хостов Nginx с использованием ролей Ansible

## Описание

Цель лабораторной работы - автоматизировать настройку нескольких виртуальных хостов **Nginx** с помощью **Ansible Roles**. Для каждого виртуального хоста создается собственная директория сайта, генерируется HTML-страница и конфигурационный файл Nginx на основе шаблонов Jinja2.

## Структура проекта

```text
.
├── inventory.ini
├── playbook.yml
├── README.md
└── roles
    └── nginx-vhosts
        ├── handlers
        │   └── main.yml
        ├── tasks
        │   └── main.yml
        └── templates
            ├── index.html.j2
            └── site.conf.j2
```

## Содержимое файлов

### `inventory.ini`

Список целевых хостов.

```ini
[web]
172.16.10.3 ansible_user=runner
```

### `playbook.yml`

Плейбук определяет список виртуальных хостов и последовательно применяет роль `nginx-vhosts` для каждого из них.

Список виртуальных хостов:

- etis.com
- fizfak.ru
- mehmat.ru

### Роль `nginx-vhosts`

Роль выполняет следующие действия:

1. Устанавливает веб-сервер **Nginx**.
2. Создает каталог сайта `/var/www/<имя_сайта>`.
3. Генерирует страницу `index.html` из шаблона Jinja2.
4. Создает конфигурационный файл виртуального хоста в `/etc/nginx/sites-available/`.
5. Активирует виртуальный хост, создавая символическую ссылку в `sites-enabled`.
6. Удаляет стандартную конфигурацию `default`.
7. Перезапускает сервис **Nginx** при изменении конфигурации.

### Шаблоны

#### `index.html.j2`

Создает HTML-страницу следующего вида:

```html
<h1>Hello from {{ vhost }}</h1>
```

#### `site.conf.j2`

Создает конфигурационный файл виртуального хоста Nginx с использованием имени сайта и соответствующей директории.

## Запуск

Выполнить команду:

```bash
ansible-playbook -i inventory.ini playbook.yml
```

## Результат

После успешного выполнения плейбука:

- установлен веб-сервер **Nginx**;
- созданы каталоги сайтов для каждого виртуального хоста;
- сгенерированы HTML-страницы из шаблона;
- созданы и активированы конфигурации виртуальных хостов;
- удалена стандартная конфигурация Nginx;
- сервис **Nginx** перезапущен для применения изменений.
